## [0.11.7] - 2026-04-09

### Added
- Prescription PDF generation via WeasyPrint matching original clinic letterhead exactly
- Compressed JPEG assets extracted from original template (~255KB total, ~365KB per PDF)
- Paper texture background, watermark, 3 arabesques, stamp/cairo emblems, transparent logo
- Standalone /prescriptions page with create, finalize, void, download PDF
- WhatsApp sharing: Web Share API sends actual PDF file (mobile), download fallback (desktop)
- Email sharing: Gmail SMTP (maadiclinic.noreply@gmail.com) with PDF attachment
- Instruction PDF on clinic letterhead (no patient info, bilingual, flowing footer)
- WhatsApp + Email buttons on instruction preview dialog (icon-only buttons)
- Real Maadi Clinic logo on login page, header bar, favicon
- Settings gear dropdown for User Management + Audit Log (frees dashboard space)
- Centered dashboard card layout (2 rows: 4 + 3)
- Browser tab: "Maadi Clinic", PWA manifest updated
- Medication dropdown limit raised from 200 to 2000 (all 1,749 drugs available)
- Prescriptions nav pill in CardLayout
- Prescriptions page lazy-loaded with Suspense (blank page mitigation)
- Dialog component: max-height 80vh with scrollable content

### Fixed
- Prescription PDF was 12MB (raw PNG assets) — compressed to ~365KB
- Instruction PDF empty first page — switched to flowing footer
- Dialog footer buttons cut off on small screens — constrained dialog height
- Prescriptions page blank on first SPA navigation — useEffect rewrite + lazy load
- Email endpoint now accepts email_override from dialog input
- Patient separator matches header separator (2px green)
- TypeScript build errors (unused imports, variables, JSX syntax)

### Changed
- "Rx" pill nav label → "Prescriptions"
- "CardioClinic" in browser tab/manifest → "Maadi Clinic"
- Email template says "Maadi Clinic" not "CardioClinic"
- Removed unauthorized "Generated by Maadi Clinic" from PDF footer

## [0.8.0] - 2026-04-07

### Added
- **Global Admin role** — full system access, purple badge
- Role-based access control (RBAC): receptionist sees only Appointments, nurse sees clinical pages, doctor sees everything, admin sees everything + user management
- Email field in Edit User dialog
- Password reset field in Edit User dialog ("New Password" — leave blank to keep current)
- Delete User button with confirmation dialog (permanent delete, not just deactivate)
- Alembic migration for admin role enum and column widening (phone varchar 20→50, smoking_packs_day numeric 3,1→5,1)

### Access Matrix
| Role | Access |
|---|---|
| Receptionist | Dashboard, Appointments |
| Nurse | + Patients, Medications, Instructions, Risk Calc, Drug Interactions |
| Doctor | + User Management, Audit Log, Prescriptions (write) |
| Admin | Full access to everything |

## [0.7.1] - 2026-04-07

### Fixed
- **Blank patient detail page** — `crypto.randomUUID()` crashes on HTTP (non-HTTPS) contexts. Replaced with Math.random fallback in toast.tsx and ClinicalForms.tsx
- **All dialog save buttons not working** — same root cause: toast crash prevented success callback from executing, making dialogs appear stuck
- Added Delete User button and confirmation dialog in User Management
- Deactivate button styled amber, delete button styled red
